## The Shift in Vision

Sūrya Nagari is one city. One blueprint. One location in India.

**Satyug — The Collective** is the platform that lets anyone, anywhere build *their* version of it.

You don't move to Sūrya Nagari. You open the app. You find land near your city. You join or start a Collective. You build your home. And together — with your neighbours — you own everything that makes life free.

This is not a real estate app. It is the infrastructure for the golden age.

## How It Works (The User Journey)

```
1. OPEN THE APP
   → See a map of your city and its outskirts
   → Every available land parcel shown with live owner-set price
   → Prices pulled from latest known government circle rates + owner listings

2. CHOOSE YOUR LAND
   → Filter by: city, distance from centre, size, price, terrain, zone
   → See the land's Collective status:
        - "No Collective yet — start one" (you become Founding Member)
        - "Collective forming — 12/50 families joined" (join now)
        - "Collective active — accepting new members" (join, facilities listed)

3. JOIN OR START A COLLECTIVE
   → A Collective is a group of families who co-own shared infrastructure
   → Minimum viable: 50 families → unlocks: Solar + Kitchen + Health
   → 200 families → unlocks: GPU Campus + University + EV Fleet
   → 1,000+ families → unlocks: Lake + Olympic Zone + Full Hospital
   → You choose which facilities your Collective votes to build first

4. BUILD YOUR HOME
   → Submit: plot size, home type (1BHK / 2BHK / 3BHK / custom), style, budget
   → Verified GCs respond with real-time quotes (price, timeline, materials)
   → You compare, choose, sign — contract tracked on the app
   → Reviews after completion: both sides, verified, immutable

5. LIVE
   → Your Collective's shared infrastructure comes online as it's built
   → Free electricity from your solar farm
   → Free meals from your community kitchen
   → Free health consult at your wellness centre
   → Free schooling for your children
   → You get paid: GPU revenues + any commercial surplus → dividend to all members
```

## The Pricing Model

**Land:** Set by the owner. Live. No algorithm. Chronological listing.
Owner receives 100% of the land sale price. No platform commission on land.

**Collective Share:** Your contribution to shared facilities. Calculated as:
```
Your share = (Total collective facility cost) ÷ (Number of member families)
```
Transparent. Voted on by members. No hidden fees.

**Home Construction:** Real-time GC quotes. You pay the GC directly.
Platform charges GC a flat subscription fee (not a % of each job).
No commission from consumers. Ever.

**Platform Revenue:** GC subscriptions + Collective admin fee (small flat annual fee per active Collective) + optional featured placement for GCs (clearly labeled, not a ranking boost).

## The No-Algorithm Rules (Same as Home App — Non-Negotiable)

| Rule | How |
|---|---|
| Land sorted by `listed_at` only | No ranking. No "hot in your area." |
| GCs sorted alphabetically or by user filter | No recommendation engine. |
| Owner sets land price | App never suggests. No "price to sell faster." |
| Collective facilities chosen by member vote | App facilitates the vote. Never recommends. |
| Reviews = verified contracts only | No unverified review. Not deletable. |
| Paid GC visibility = labeled banner | Fixed slot, clearly marked FEATURED. Not a rank change. |

## The Collective Governance Model

Every Collective runs on the same governance as Sūrya Nagari:

- **1 home = 1 vote** on all Collective decisions
- Proposals submitted by any member
- AI prepares unbiased data brief (cost, impact, simulation) — no recommendation
- All members vote (app-based, blockchain-recorded, immutable)
- Majority decides; AI implements operational changes

**What members vote on:**
- Which facility to build next
- Budget allocations
- New member admissions (optional — open Collectives accept all)
- Land use changes
- Surplus distribution (how much reinvest vs dividend)

## The Satyug Collective Constitution
### (Every Collective agrees to these at formation)

1. **Pure Vegetarian** — Community kitchen, farms, and shared spaces are 100% vegetarian. Private homes are private.
2. **Non-Violence** — Physical violence = forced buyout of your share at 70% book value.
3. **Open Books** — Every rupee of collective spending is on a public ledger all members can read.
4. **Non-Speculative** — Collective shares cannot be sold above inflation + improvements. ROFR by Collective.
5. **Democratic** — Every decision by member vote. No appointed leaders. Rotating coordination roles.
6. **Open Source** — The Collective's plan, legal templates, and AI systems are published freely. Fork it.
7. **Ecological** — No chemical farming. No animal harm in shared spaces. Trees protected by charter.
8. **Inclusive** — No discrimination by religion, caste, gender, age, or origin. One human standard.

## Technology Architecture (The App)

### Data Model

```
COLLECTIVE
  id · name · location_geo · status: forming|active|mature
  member_count · facilities: Facility[]
  treasury_balance · monthly_surplus · governance: Vote[]

LAND_PARCEL
  id · owner_id · collective_id (if assigned)
  size_sqft · price_inr (owner-set) · status
  geo_coords · circle_rate_inr (govt reference, auto-pulled)

MEMBER (= OWNER after joining)
  id · collective_id · home_id
  share_fraction · voting_record · review_history

HOME
  id · parcel_id · member_id · type: 1BHK|2BHK|3BHK|custom
  sqft · status: planned|quoted|building|complete
  contract_id

GC (General Contractor)
  id · name · verified · specialization · portfolio[]
  avg_rating · subscription_tier

QUOTE
  id · home_id · gc_id · price_inr · timeline_months
  materials_grade · status: pending|accepted|rejected

CONTRACT → REVIEW (verified, immutable)

FACILITY
  id · collective_id · type (from the 30-facility menu)
  status: voted|funded|building|operational
  cost_inr · funded_by: member_contributions[]

VOTE
  id · collective_id · proposal_text
  ai_brief_url · options[] · results[] · status: open|closed
  blockchain_hash (immutable record)

CIRCLE_RATE (reference data)
  state · district · zone · rate_inr_per_sqft · as_of_date
  source: govt_registry_api
```

### Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Mobile App | React Native | iOS + Android from one codebase |
| Web App | React + Vite + Tailwind | Matches existing stack |
| Backend | FastAPI (Python) | Async, proven, SSE support |
| Database | PostgreSQL + PostGIS | Geo queries, land parcel maps |
| Real-time | Server-Sent Events (SSE) | GC quote delivery, vote results |
| Maps | MapLibre GL JS | Open source, no Google Maps fees |
| Govt. data | State land registry APIs | Live circle rates where available |
| Governance | On-chain hash via Polygon | Vote immutability, low gas cost |
| Auth | Phone OTP | India-first, no email required |
| Deploy | Railway | Existing platform, auto-deploy |
