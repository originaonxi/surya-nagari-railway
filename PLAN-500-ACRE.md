> derived from the **same sourced rates already in `prospectus.html`** (InfraLens 2026,
> MGetEnergy 2026, AgriJob.in 2026, IREDA 2026, NVIDIA 2026). No new sources invented.
> Until a real DPR is commissioned, all cost and revenue figures are indicative only.

---

## Why Two Projects?

The 5,000-acre vision in `index.html` + `prospectus.html` stands unchanged.
The 500-acre prototype (`500/index.html`, new) is its **living proof** — smaller, denser,
buildable in 3–4 years — that proves every system works before the full scale begins.
It also provides real land and real owners for the **home-building marketplace app**.

## PART 2 — THE 50-BED WELLNESS CENTRE (Not a Hospital)

**What it is:** A multi-disciplinary consultation and short-stay wellness hub.
**What it is not:** No emergency room, no surgery wing, no ICU.
The 50 beds are for post-therapy rest, monitoring, and short observation stays.
All disciplines under one roof; AI-assisted triage routes residents across traditions.

| Discipline | Spaces |
|---|---|
| **Allopathy** | 4 consultation rooms, diagnostics lab, pharmacy |
| **Ayurveda** | 6 consultation rooms, Panchakarma wing, herb dispensary |
| **Nutrition Science** | 3 consultation rooms, metabolic lab, diet planning studio |
| **Yoga Therapy** | 2 group halls, 4 private rooms, outdoor deck |
| **Massage & Bodywork** | 6 rooms (Abhyanga, Swedish, Thai, Deep Tissue) |
| **Traditional Chinese Medicine** | Acupuncture suite, herbal dispensary, moxibustion room |
| **Tribal / Indigenous Medicine** | 2 consultation rooms, living medicinal plant walk |
| **Mental Wellness** | 4 therapy rooms, 1 group circle, silent retreat pod |
| **Diagnostics** | Blood lab, ECG, X-ray, DEXA, body composition |

**How it works:** Resident books a 30-min intake. AI routes to the right discipline(s).
Cross-referral across traditions is built-in. Cost to resident: **₹0** (Sovereign Baseline).

## PART 3 — THE HOME-BUILDING MARKETPLACE APP

### Concept

"Uber for building your dream home — on land where survival is already free."

The city eliminates survival anxiety (free electricity, water, food, health, internet).
The app eliminates home-building anxiety: transparent GC quotes, owner-only land,
zero algorithm, 100% authentic reviews.

### Core Domain Model

```
OWNER (verified MSCS shareholder)
  └─ lists LAND PARCEL (sale or rent — owners only, no corporates)
       └─ CONSUMER submits HOME REQUIREMENTS {size, style, budget_range}
            └─ verified GC responds with real-time QUOTE {price, timeline, materials}
                 └─ accepted QUOTE becomes CONTRACT {milestones, payments}
                      └─ completed CONTRACT unlocks REVIEW {rating, text, immutable}
```

### Entities

**LAND_PARCEL**
`id · owner_id · parcel_number (co-op registry) · size_sqft · zone · geo_coords`
`utilities_connected: always true (solar + water + fiber pre-built by city)`
`status: available | listed | under_contract | sold | building`

**OWNER**
`id · share_id (verified against MSCS registry) · name · phone`
`verified: true (no corporate accounts, no unverified listings)`

**CONSUMER**
`id · name · email · phone · kyc_status`
`→ becomes OWNER after purchase`

**GC (General Contractor)**
`id · name · years_experience · specialization · portfolio_images[]`
`insurance_verified · city_registry_verified · avg_review_score`

**QUOTE**
`id · consumer_id · parcel_id · gc_id`
`home_type: 1BHK | 2BHK | 3BHK | custom · sqft`
`price_inr · timeline_months · materials_grade: standard | premium | luxury`
`status: pending | accepted | rejected | expired`

**CONTRACT**
`id · quote_id · signed_date · milestone_checklist[] · payment_schedule[]`

**REVIEW**
`id · contract_id · reviewer_id · reviewee_id`
`rating: 1–5 · text (mandatory) · verified_purchase: true · immutable: true`
`→ cannot be deleted by reviewed party`
`→ cannot be posted without a valid completed contract_id`

### No-Algorithm Rules (Hard Constraints, Not "Settings")

| Rule | Implementation |
|---|---|
| Listings sorted by `listed_at DESC` only | No ranking model. No "relevance" score. |
| GCs sorted alphabetically or by user-chosen filter | No recommendation engine. |
| Owner sets price — app never suggests | No "competitive pricing nudge" ever. |
| User filters: price_range, zone, size, gc_rating, timeline | App applies the filter exactly as set. Nothing else. |
| Paid visibility = labeled fixed banner only | A paid GC gets one `FEATURED` slot clearly marked. Never a search rank change. |
| Reviews need verified `contract_id` | No review = no verified transaction. No exceptions. |
| Reviews cannot be suppressed | Reviewed party cannot flag to remove. Only court order. |

### App Screens — Scope 1.0

1. **Land Map** — OpenStreetMap/MapLibre view of available parcels + list view
2. **Parcel Detail** — Size, zone, owner profile, utilities, photos
3. **Build My Home** — Submit requirements → GC quotes appear in real time (SSE)
4. **GC Profiles** — Portfolio, verified reviews, insurance, specialization
5. **Owner Dashboard** — Manage parcels, view inquiries, track active contracts
6. **Consumer Dashboard** — Active quotes, accepted contracts, milestone tracker
7. **Reviews** — Browse all reviews for a GC or owner; leave verified review

### Proposed Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Frontend | React + Vite + Tailwind | Matches c1api / Omnisail existing pattern |
| Backend | FastAPI (Python) | Async, proven in this codebase |
| Database | PostgreSQL + PostGIS | Geo queries for parcel map |
| Auth | Phone OTP — India-first | Matches co-op resident registry |
| Real-time quotes | Server-Sent Events (SSE) | Consistent with c1api streaming |
| Maps | MapLibre GL JS (open-source) | No Google Maps cost |
| Deploy | Railway (existing) | One platform, auto-deploy on git push |
